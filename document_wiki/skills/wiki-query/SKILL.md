# Wiki Query

## Назначение

Используй этот skill, когда нужно ответить на вопрос пользователя по базе document_wiki.

QueryDeepAgent использует wiki как карту поиска, но финальный ответ должен строиться по файлам из `sources/`.

## Основной принцип

```text
Wiki помогает найти релевантные source-файлы.
Sources подтверждают ответ.
```

Не используй wiki как окончательный источник фактов. Wiki может быть неполной или краткой.

## Последовательность работы

1. Получи вопрос пользователя.
2. Прочитай `wiki/index.md`.
3. По navigation notes и вопросу выбери релевантные dimensions.
4. Прочитай только релевантные `wiki/dimensions/*.md`.
5. Определи релевантные source-файлы.
6. Прочитай source-файлы из `sources/`.
7. Сформируй ответ по source-файлам.
8. Укажи, из каких source-файлов взята информация.
9. Если данных нет, прямо скажи, что информация не найдена в текущей базе документов.

## Формат ответа

```markdown
## Ответ
...

## Источники
- `sources/doc_001.md`
- `sources/doc_003.md`

## Ограничения
...
```

Если ответ не найден:

```markdown
## Ответ
Информация не найдена в текущей базе документов.

## Проверенные источники
- ...

## Ограничения
...
```

## Запрещено

- Редактировать `wiki/`.
- Создавать новые dimensions.
- Редактировать `sources/`.
- Использовать прошлые ответы как источник знаний.
- Отвечать без опоры на source-файлы.
- Использовать vector search, RAG, BM25, reranker или локальные индексы.
- Выдавать информацию из wiki как окончательный источник истины.
