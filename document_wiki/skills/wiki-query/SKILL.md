# Wiki Query

## Назначение

Используй этот skill, когда нужно ответить на вопрос пользователя по базе document_wiki.

QueryDeepAgent использует wiki как карту поиска, но финальный ответ должен строиться по файлам из `sources/`.

Пользователь может задать короткий запрос: одно слово, фамилию, аббревиатуру, название
сервиса, технологии, проблемы или метрики. В таких случаях используй `Search terms`,
`Key topics`, `Potential questions`, `Navigation notes` и source-ссылки внутри wiki.

## Основной принцип

```text
Wiki помогает найти релевантные source-файлы.
Sources подтверждают ответ.
```

Не используй wiki как окончательный источник фактов. Wiki может быть неполной или краткой.

## Последовательность работы

1. Получи вопрос пользователя.
2. Прочитай `wiki/index.md`.
3. По navigation notes, search terms, source summaries и вопросу выбери релевантные dimensions.
4. Если запрос короткий или неоднозначный, прочитай несколько потенциально релевантных dimensions.
5. В dimension-файлах ищи точные совпадения, синонимы, аббревиатуры, русские и английские варианты.
6. Определи релевантные source-файлы.
7. Прочитай source-файлы из `sources/`.
8. Сформируй ответ по source-файлам.
9. Укажи, из каких source-файлов взята информация.
10. Если данных нет, прямо скажи, что информация не найдена в текущей базе документов.

## Формат ответа

```markdown
## Ответ
...

## Источники
- `sources/doc_001.md`
- `sources/doc_003.md`

## Ограничения
...
```

Если ответ не найден:

```markdown
## Ответ
Информация не найдена в текущей базе документов.

## Проверенные источники
- ...

## Ограничения
...
```

## Запрещено

- Редактировать `wiki/`.
- Создавать новые dimensions.
- Редактировать `sources/`.
- Использовать прошлые ответы как источник знаний.
- Отвечать без опоры на source-файлы.
- Использовать vector search, RAG, BM25, reranker или локальные индексы.
- Выдавать информацию из wiki как окончательный источник истины.
