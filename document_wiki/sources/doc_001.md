# Тестовый отчет по клиентским обращениям за июнь

## Распаршенный текст

Отчет подготовлен для проверки качества обработки клиентских обращений в июне.
Автор отчета: Иванов Иван. Владелец блока по метрикам: Петров Петр.

В документе рассматриваются обращения клиентов по банковским картам, переводам и мобильному приложению.
Основные метрики отчета: CSI, NPS, количество обращений, доля повторных обращений и среднее время решения проблемы.

По банковским картам чаще всего встречаются жалобы на отклонение операций, задержку уведомлений и спорные списания.
По переводам клиенты сообщают о задержках зачисления и сложностях при проверке статуса платежа.
По мобильному приложению отмечены проблемы с авторизацией, отображением истории операций и нестабильной работой push-уведомлений.

Отдельный раздел посвящен процессу контроля качества. Команда поддержки проверяет обращения с низким CSI,
выделяет повторяющиеся причины недовольства и передает продуктовым командам список проблем для исправления.

В отчете указано, что по операциям с высоким фрод-риск score применяется дополнительная проверка.
Срабатывания фрод-мониторинга анализируются отдельно, чтобы отделить корректные блокировки от ложноположительных срабатываний.

Рекомендации отчета:

- обновить сценарии информирования клиентов по отклоненным операциям;
- улучшить отображение статуса переводов в мобильном приложении;
- добавить отдельный контроль повторных обращений по картам;
- анализировать ложноположительные срабатывания фрод-мониторинга;
- передавать продуктовым командам еженедельный список клиентских проблем.
