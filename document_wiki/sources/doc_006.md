# Аудит качества данных витрины клиентских обращений

## Распаршенный текст

Документ подготовлен командой Data Quality. Автор: Морозова Елена.
Участники проверки: команда DWH, владельцы витрины клиентских обращений и аналитики клиентского опыта.

Аудит проверяет корректность данных в витрине `dm_customer_requests`.
Основные источники данных: CRM, контакт-центр, мобильное приложение, система претензий и справочник продуктов.

Проверяемые поля:

- request_id;
- client_id;
- product_code;
- request_channel;
- request_reason;
- created_at;
- closed_at;
- csi_score;
- operator_team;
- escalation_flag.

Основные метрики качества данных: полнота заполнения, доля дублей, доля некорректных product_code, расхождение количества обращений между CRM и витриной, задержка обновления данных и количество записей без закрытия.

Обнаруженные проблемы:

- часть обращений из мобильного приложения приходит без корректного `request_reason`;
- в CRM и витрине отличаются значения каналов для обращений из чата;
- по некоторым продуктам отсутствует связь со справочником продуктов;
- есть дубли обращений при повторной передаче события из контакт-центра;
- `closed_at` иногда меньше `created_at` из-за ошибок часового пояса.

Рекомендации:

- создать регулярные data quality checks;
- добавить контроль уникальности `request_id`;
- нормализовать справочник каналов обращений;
- сверять количество обращений между CRM и витриной ежедневно;
- выделить владельца поля `request_reason`.
