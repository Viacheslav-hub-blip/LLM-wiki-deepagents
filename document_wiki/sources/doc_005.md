# Отчет по SLA обработки внутренних инцидентов

## Распаршенный текст

Документ подготовлен офисом операционной устойчивости. Автор: Васильев Олег.
Владельцы процесса: команда Incident Management и команда Platform Operations.

Отчет описывает внутренние инциденты за третий квартал: сбои интеграций, недоступность внутренних сервисов, задержки пакетных обработок и ошибки обмена с внешними провайдерами.

Основные метрики:

- количество инцидентов P1, P2 и P3;
- среднее время обнаружения инцидента;
- среднее время восстановления сервиса;
- доля инцидентов, закрытых в SLA;
- количество повторных инцидентов по одной причине.

Ключевые процессы:

- регистрация инцидента в системе управления инцидентами;
- первичная triage-оценка влияния;
- эскалация в ответственную команду;
- коммуникация статуса заинтересованным подразделениям;
- postmortem-разбор после критичных инцидентов.

Основные проблемы:

- не всегда указан владелец восстановления сервиса;
- часть инцидентов закрывается без root cause;
- повторные инциденты не всегда связываются с предыдущими postmortem;
- в коммуникациях не хватает единого статуса для бизнес-подразделений.

Рекомендации:

- создать dimension для SLA и инцидентов, если такого разреза еще нет;
- вести отдельный список повторных root cause;
- добавить контроль заполнения postmortem;
- измерять время до первой коммуникации по P1 и P2.
